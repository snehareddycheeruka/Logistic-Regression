1.Why we need to move from Linear to Logistic??

1. Linear Regression predicts continuous values.

2. Examples:

   * House Price Prediction
   * Salary Prediction
   * Marks Prediction

3. But classification problems need outputs like:

   * Yes / No
   * True / False
   * 0 / 1

4. Linear Regression may predict values like:

   * 1.8
   * -2.5
   * 0.8

5. These values do not make sense for classification problems.

6. So we need a model that gives output only between 0 and 1.

7. Logistic Regression uses Sigmoid Function to convert values into probabilities.

8. If probability > 0.5 → Class 1

9. If probability < 0.5 → Class 0

10. That is why we use Logistic Regression.

--------------------------------------------------------------------------
2.Why it is called Logistic Regression??


Logistic Regression uses a regression equation internally:

z = mx + c

where:
m = slope or weight
x = input feature
c = intercept or bias

Then it applies a special function called:

Sigmoid Function:

z = 1/(1+e^(-z))

This converts values from:
-infi to +infi

into:
0 to 1

The sigmoid function changes large values
into probability values between 0 and 1.

If output > 0.5 → Class 1
If output < 0.5 → Class 0

Hence it performs classification using regression mathematics.

That is why it is called Logistic Regression.


3.Math behind Logistic Regression

Logistic Regression first uses a linear equation:

z = mx + c

where:
m = slope
c = intercept
x = input value

m decides how much the input affects the output.
c shifts the line left or right.

Example:

m = 2
c = -6

Then:

z = 2x - 6

If x = 4

z = (2 * 4) - 6
z = 8 - 6

This z value is then passed into the Sigmoid Function
to convert it into a probability value between 0 and 1.

Confusion Matrix:

Confusion Matrix is used to check the performance of a classification model.

It compares Actual values with Predicted values.

1 = Positive
0 = Negative

Types in Confusion Matrix:

1. True Positive (TP)
Actual = 1 and Predicted = 1

2. False Negative (FN)
Actual = 1 and Predicted = 0

3. False Positive (FP)
Actual = 0 and Predicted = 1

4. True Negative (TN)
Actual = 0 and Predicted = 0

Confusion Matrix

Actual    Predicted    Result

1         1            True Positive
1         0            False Negative
0         1            False Positive
0         0            True Negative

-------------------------------------------------------------------------


Evaluation Metrics

Evaluation Metrics are used to measure the performance of a classification model.

1. Accuracy

Accuracy checks overall correct predictions.

Formula:

Accuracy = (TP + TN) / (TP + TN + FP + FN)

2. Precision

Precision checks how many predicted positive values are actually correct.

Formula:

Precision = TP / (TP + FP)

3. Recall

Recall checks how many actual positive values are correctly predicted.

Formula:

Recall = TP / (TP + FN)

4. F1 Score

F1 Score balances Precision and Recall.

Formula:

F1 Score = 2 * Precision * Recall / (Precision + Recall)

Train Logistic Regression Model

After splitting the dataset,
the Logistic Regression model is trained.

The model learns patterns from the training data.

x_train → Input features for training
y_train → Output labels for training

The fit() function is used to train the model.

Here:

into a 1-dimensional array.

Formula:

the z value into probability.

If probability is high:
Prediction → Pass

If probability is low:
Prediction → Fail
